~ Gaussian Sonificator - Visualisator ~

This page presents the interactive audio-visual application system “Gaussian Sonificator - Visualisator” both in the general context of practical use and in that of the technical details and information about the tools & methods required for the development of an interactive audio - visual application system. The research and finally the implementation of the aforementioned idea is a manifestation of the relationship between art and technology and the strong tendency that exists in recent years regarding interaction, especially in audiovisual systems. The methods related to the process of developing and adapting the programming tools that have been selected are then presented, as well as additional information related to the justification for the selection of the specific tools. Additionally, the source code will be presented and explained. Finally, the artistic expectations of the creator are presented as well as some comments on the final result and its usability.

Setup

Firstly, load up the layout on the touch-osc app (Android). Copy the .touchosc file to your Android smartphone and start the TouchOSC application. Go to Layout => Add from file (add Gaussian-Sonification-Visualisator.touchosc). Set up the host address & port (computer's IP address & 57120 default incoming OSC messages port from SuperCollider)

You can get it from my GitHub repo.

Get Touch-osc layout (GitHub)


Now in SuperCollider, make sure that the IP address is identical with your computer's IP. To check your IP from the shell, just type the following (macOS 10.13.6):

Shell

ipconfig getifaddr en0
`#for older MacBook Pros use en1` \


SuperCollider

//TouchOSC-SuperCollider ~n = NetAddr("192.168.1.6", 57120); //SuperCollider - openFrameworks
~nsend = NetAddr("192.168.1.6", 12345);


After the appropriate changes evaluate the scd file.


Then, create a new project using the projectGenerator app in openFrameworks and add "ofxOsc" in the addons section. Navigate to the project directory and replace the "src" directory with the one from the GitHub repo (subdirectory inside OF directory)

Type the following code in the shell to compile.

Shell

make -f Makefile
#then
make RunRelease

Developing process

SuperCollider code

The following code block was used to set up the gaussian distribution in SuperCollider. The ".gauss" method has two (2) arguments, gaussian receiver and standard deviation. These parameters are defined as environment variables which are updated with each OSC message they receive from the Android smartphone interface. The gaussian distribution (.gauss) produces both positive and negative values. If a negative values comes up is immediately multiplied with -1.

~gss = gauss(~receiver, ~stdevi).round(0.00001);
if (~gss.isNegative, {~gss = ~gss * (-1)}); ~gss.postln;



Now, the OSC protocol should be set. Both in SuperCollider and openFrameworks (openFrameworks C++ code). On the SuperCollider side (OSC-server) regarding the smartphone - SuperCollider communication the NetAddr class was used. SuperCollider receives incoming messages from the 57120 port by default. NetAddr class "hostname" argument => device's IP address.

//TouchOSC-SuperCollider
~n = NetAddr("192.168.1.6", 57120); //SuperCollider - openFrameworks`
~nsend = NetAddr("192.168.1.6", 12345);

OSC communication was tested using a simple OSCdef.

//Test Gaussian Function Receiver OSCdef(\receiver, {
|msg, time, addr, port|
msg.postln;
},'/gauss/receiver');



The whole application, regarding sound, is based on a continuous Routine (fork) which stops only if the OFF button is pressed from the interface. More in the source code.



There are two OSCdefs for the expression of the red sliders in SuperCollider. These two OSCdefs:
1. Receive new values from the "receiver" & "standard deviation" sliders
2. Update the corresponding environment variables with the new values

There is also a third OSCdef which determines the generation time (in seconds) of a new value.

//Gaussian Function Receiver
OSCdef( \receiver, {
|msg, time, addr, port|
if(
~onoff == 1, {
~receiver = msg[1].round(0.00001).linlin(0.0, 1.0, 0.0, 100);
["Receiver:", ~receiver].postln;
});
},'/gauss/receiver');


Additionally, there are four OSCdefs that each one responds to the ON/OFF messages sent from Synth 1, 2, 3, 4 buttons. Practically, they are turning the SynthDef's volume up or down by registering the "0.0" value to the amplitude argument of each SynthDef for "OFF" mode and a non zero value for "ON" mode.

The following code expresses the aforementioned function. It is actually the code responding to the messages from the Synth 1 button. A Pdef was also been used (amplitude & duration variations).

//Synths-ON-OFF
//Synth1
OSCdef(\s1onoff, {
|msg, time, addr, port|
~s1onoff = msg[1];
if(
Main ON-OFF
~onoff == 1,{
if(
if Synth 1 => ON
~s1onoff == 1, {
Pdef(\noisedeff).set(
\amp, Pwhite(0.33, 0.66));
Synth("noisedef",[\amp, 0.5]).set;
"Synth 1 ON".postln;
});
if(
if Synth 1 => OFF
~s1onoff == 0, {
Pdef(\noisedeff).set(\amp, 0.0);
Synth("noisedef", [\amp, 0.0]).set;
"Synth 1 OFF".postln;
});
});
}, '/gauss/synth1');

Finally, the values generated by the gaussian function should be sent to openFrameworks using the ~nsend address (port 12345). The following line of code is placed in the routine.

~nsend.sendMsg('/gauss-value', ~gss.round(1));

Then a "for" loop is created in the draw() function. Inside the for loop a colour function is defined. In this case the colour will continuously change (using a sine, noise & Random) according to the new gaussian function value (received from SuperCollider). The ofRotateDeg function is used for rotation. The ofScale function follows so that "levels" are created among shapes in each subsequent cycle of the "for" loop (where a new shape is created). Then the ofDrawCone function follows so that a cone shape can be drawn. The xcor, yxor, initialval, blue and red variables are derived from the received gauss value. More information will be reported later.

for (int i = 0; i < 100; i++) {
ofSetColor(red * sin(
cos(initialval * 0.01) * i * 0.02) + (
sin(i * 0.01)),
ofNoise(initialval * 2) * ofRandom(5.0, 10.0),
sin(i * 0.01) * blue - (blue/2 * 0.5));

ofRotateDeg(ofGetElapsedTimef() + (initialval * 0.1)); //* -1 for backwards

ofScale(0.88);

ofDrawCone(ofClamp(xcor, 50.0, 200.0),
ofClamp(ycor, 50.0, 200.0),
200, 200);
}


In order that the colour & spatial variables are going to change continuously another "for" loop was created. Every time that a new value is received these variables are updated, however this is not done instantly. The values registered to the variables are increased or decreased "step by step" until they reach the received OSC message's value. The loop will perform the function and the "if" conditional allows the algorithm to recognise if the OSC value that is going to be set is less than or greater than the previous received value. This allows the colour variables and the spacial ones to always change starting from the value they had before the new OSC message.
